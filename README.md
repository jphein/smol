# smol

A whole game console ‚Äî and a pocket watch ‚Äî on a **$3 ESP32-C3 SuperMini with a 0.42" (72√ó40) OLED**.

It started as *"can we make this into a tiny game player? can it run Minecraft?"* Answer: **real Minecraft, no** (400 KB RAM vs. gigabytes) ‚Äî **but the soul of it, yes.** From there it grew into a little platform: multiple games, a unified Rust firmware with a menu, an ESP-NOW mesh (now with **time sync** and **magical node names**), a 3D-printable pocket-watch case, and a self-hosted project site. Now on hardware: all three boards run the full stack on **build 58** ‚Äî the shared-world **MMO snake** with six **treasure-powers**, a **plugin framework** (About screen, scrolling menu, Bench mesh-view), the sigil **version name on the boot splash**, and a **native Home Assistant integration** over MQTT: battery **voltages + SOC** and **grid power** on every display, a **HA node manager** that sets each board's default screen remotely (gateway-self **shipped + verified on glass**, plus **leaf-relay** over the mesh), and a git-ignored **per-board config file** (#19). The **everything-release** (#12/#21/#25/#27/#33) then added **typed HA entities** (one device per node), a **WLED WiZmote** physical remote, a live **mesh-topology card**, and a native **HA Update entity** ‚Äî the five **hardware-verified fleet-wide** (id7 5/5 + all three boards on build 58, USB). The **ESP-NOW‚Üíinternet relay** is proven end-to-end; **OTA updates** ‚Äî engine, publish tooling and HA panel are **built and deployed**, and **OTA updates are now proven** ‚Äî after adding a firewall allow-rule so the boards can reach the image host, a canary **self-updated build 58‚Üí59 over the air in ~17 s** (fetch‚Üíverify‚Üíboot‚Üí`Valid`); üü¢ (#6, [#37](https://github.com/jphein/smol/issues/37) resolved).

The **unified Rust firmware** (`rust/clock/`, `no_std` esp-hal) ties Clock + Snake + Bench into **one binary** with a BOOT-button menu; the blue LED shows ESP-NOW peer state (off ‚Üí blink = detected ‚Üí solid = connected) in the background across all modes. Verified: two boards go **solid blue = connected**.

The mesh now also **shares the time** (a `SMOLv1 TIME` frame; a board adopts a peer's time only when the peer's NTP sync is more recent, so it's loop-free) and gives every board a **magical realm name** ported from [realm-sigil](https://github.com/jphein/realm-sigil) (display-only ‚Äî id7 *Draconic Dominion*, id8 *Eldritch Nexus*, id9 *Jade Herald*). ## The pocket watch
`experiments/pocketwatch/` generates a parametric round case ‚Äî **body + lid + crown** (3 printable STLs) ‚Äî in the classic orientation: the chain **bail** and a removable **crown that covers the USB-C port** stack at the top; the OLED, buttons and clear-PLA LED light-pipes sit below; pockets inside for the board + a 502030 LiPo + a TP4056. The OLED is rotated 180¬∞ in firmware so it reads upright when hung. Regenerate renders with `render_previews.py`.

## Quick start
See **[docs/BUILDING.md](docs/BUILDING.md)**. TL;DR: Arduino games flash with `arduino-cli` (`esp32:esp32:esp32c3`); the Rust firmware builds with `cargo build --release --features espnow` and flashes with **espflash v3**. Run the site locally with `python3 site/server.py`.
